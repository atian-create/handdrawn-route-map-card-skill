# Route Map Card Rules

Use these rules when running `$handdrawn-route-map-card`.

## Inputs To Check

- topic or route
- scene type: city food, travel itinerary, hiking, life checklist, or workflow manual
- 5-7 nodes, if known
- style preference
- whether the user wants image generation or just a brief

## Workflow Rules

- Classify the topic into a map card type.
- Build a 5-7 node route or checklist.
- Create side collage subjects, mini box, and bottom note.
- Avoid fake precise navigation, prices, hours, or official claims.
- Return a generation brief and quality checklist.
- If image generation is available, generate and review the file; otherwise return the brief.

## Output Must Include

- content structure: title, subtitle, nodes, side box, bottom note
- image-generation brief
- quality checklist
- caption pattern when needed
- notes about verification if the map contains real places or current facts

## Quality Checks

- The output must be usable from the material provided by the user.
- Recommendations must explain their reasoning.
- If evidence is weak or missing, mark it clearly.
- Keep the first result focused; do not expand into a full consulting report by default.
- Do not claim automatic publishing, guaranteed growth, guaranteed traffic, or guaranteed sales.
- Do not copy another creator's exact wording, images, story, identity, or claims.

## Public-Safe Wording

Prefer:

- workflow
- checklist
- review
- judgment
- public signal
- user-provided material
- next action

Avoid:

- private data extraction
- secret system
- guaranteed viral result
- guaranteed traffic growth
- copy-paste another creator's content
